# Reflexión Personal sobre el Desarrollo del Proyecto

## Experiencia Durante el Proyecto

La realización de este proyecto representó una oportunidad para poner en práctica los conocimientos adquiridos durante mi proceso de formación en el SENA. A través del diseño e implementación de una base de datos para una biblioteca, pude comprender mejor cómo los conceptos teóricos se aplican en situaciones reales dentro del área de desarrollo de software.

El trabajo realizado me permitió fortalecer habilidades relacionadas con el análisis de requerimientos, la organización de información y la construcción de soluciones tecnológicas orientadas a resolver necesidades específicas.

## Conocimientos Fortalecidos

Uno de los aspectos más importantes fue el aprendizaje relacionado con las bases de datos relacionales. Durante el desarrollo del proyecto reforcé conceptos como:

* Creación de tablas y esquemas.
* Uso de claves primarias y claves foráneas.
* Aplicación de restricciones para garantizar la integridad de los datos.
* Implementación de índices para mejorar el rendimiento de las consultas.
* Elaboración de consultas SQL para obtener información específica.

Asimismo, comprendí la importancia de estructurar correctamente una base de datos antes de comenzar a almacenar información, ya que un buen diseño facilita la administración y el mantenimiento del sistema.

## Uso de Herramientas Tecnológicas

El proyecto también me permitió familiarizarme con herramientas utilizadas en ambientes profesionales.

Mediante Visual Studio Code pude gestionar y organizar los scripts SQL de manera eficiente. Con Docker aprendí la importancia de trabajar con contenedores para facilitar la implementación y ejecución de servicios. PostgreSQL me permitió desarrollar y administrar la base de datos, mientras que pgAdmin facilitó la validación de los datos y la ejecución de consultas de manera visual.

El uso conjunto de estas herramientas me ayudó a comprender cómo interactúan diferentes tecnologías dentro de un proyecto de software.

## Retos y Soluciones

Durante el desarrollo surgieron algunos inconvenientes relacionados con la configuración del entorno y la inserción de datos en las tablas. Fue necesario revisar mensajes de error, validar relaciones entre registros y realizar ajustes en los scripts para garantizar el correcto funcionamiento de la base de datos.

Estas situaciones me enseñaron que los errores forman parte del proceso de aprendizaje y que la capacidad para analizarlos y resolverlos es una habilidad fundamental para cualquier desarrollador.
